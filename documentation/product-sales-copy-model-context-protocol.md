### MCP Architecture Deep Dive

#### Components

- **MCP Hosts**: Programs like Claude Desktop, IDEs, tools that access resources through MCP
- **MCP Clients**: Maintain 1:1 connections with servers
- **MCP Servers**: Expose specific capabilities through MCP protocol
- **Local Resources**: Your machine's databases, files, services
- **Remote Resources**: APIs and services available over network

#### How It Works

- Client-server architecture
- Single host can connect to multiple servers
- JSON-RPC 2.0 messaging protocol
- Stateful connections
- Capability negotiation
- User confirmation for sensitive operations

#### Server Capabilities

Servers can offer:

- **Resources**: Context and data for user or AI model
- **Prompts**: Templated messages and workflows
- **Tools**: Functions for AI to execute

#### Client Capabilities

Clients can offer:

- **Sampling**: Server-initiated agentic behaviors
- **Recursive LLM interactions**

#### Additional Features

- Configuration management
- Progress tracking
- Cancellation support
- Error reporting
- Logging
